# Implementeren van een fhir database rest api

## architecture
 - Deze architectuur is gebaseerd op de architectuur van deze nestjs api in combinatie met mongode als storage.
 - We starten met een simplee opzet, Get data to retrieve resources from the database en een Post method die na het valideren van de data naar de database zal schrijven.

## RestApi
 - we onderstueren de volgende routes:
 - GET /api/v1/patient
 - GET /api/v1/patient?_param=value
 - GET /api/v1/patient/:id
 - POST /api/v1/patient
 - PUT /api/v1/patient/:id
 - DELETE /api/v1/patient/:id

## Searchparameter
- Er zijn heel veel search parameters, laten we ons voorlopig beperken to _id, _sort
